# Milestone v0.4: Elexon Pipeline Validation

**Status:** SHIPPED 2026-05-04
**Phases:** I1-I4
**Total Plans:** 4

## Overview

Built ENTSO-E-style validation for Elexon: inventory alignment, mocked request-shape
coverage, fixture-backed bronze-to-silver checks, opt-in live API-to-silver tests,
and user-facing CLI/backfill live smoke tests against isolated temp paths.

## Phases

**Requirements:** ELEXON-INV-01, ELEXON-INV-02, ELEXON-INV-03
**Plans:** 1 plan

Plans:

- [x] `I1-01-PLAN.md` - Elexon inventory contract and live-test scaffolding

### Phase I2: Elexon mocked request-shape and fixture-backed bronze-to-silver tests

**Requirements:** ELEXON-MOCK-01, ELEXON-MOCK-02, ELEXON-MOCK-03
**Plans:** 1 plan

- [x] `I2-01-PLAN.md` - Elexon mocked request shape and fixture-backed bronze-to-silver coverage

**Details:**

### Phase I3: Elexon live API to silver test suite

**Requirements:** ELEXON-LIVE-01, ELEXON-LIVE-02, ELEXON-LIVE-03, ELEXON-LIVE-04, ELEXON-LIVE-05
**Plans:** 1 plan

Plans:

- [x] `I3-01-PLAN.md` - Elexon live API-to-silver integration suite

**Details:**

- Opt-in live tests call the public Elexon Insights API for active configured datasets with narrow deterministic windows.
- Representative live responses are written through `BronzeWriter`, transformed to silver parquet, and checked for rows, columns, data provider, and schema validity.
- Empty/no-data responses and known removed endpoints are classified explicitly as skip/deferred/documented outcomes.
- Live tests require no API key and remain excluded from normal test runs.

### Phase I4: Elexon CLI/backfill live smoke tests and milestone close-out docs

## Milestone Summary

**Key Decisions:**

- Use config, endpoint definitions, and silver transformer registrations as the Elexon inventory contract rather than duplicating dataset lists.
- Keep removed, duplicate, or unstable Elexon endpoints visible through `EXCLUDED_ENDPOINTS`.
- Use fixture-backed tests for transformer confidence and opt-in live tests for public API compatibility.
- Runtime `GRIDFLOW_DATA_DIR`, `GRIDFLOW_DUCKDB_PATH`, and `GRIDFLOW_LOG_DIR` overrides must beat YAML paths so live CLI checks can isolate outputs.

**Issues Resolved:**

- Corrected `boal` config alignment to the active BOALF replacement endpoint.
- Added mocked request-shape coverage for all active Elexon datasets.
- Added live public API-to-silver coverage for representative Elexon request styles.
- Fixed runtime config precedence so CLI smoke tests do not write to normal project data when `GRIDFLOW_*` paths are provided.

**Issues Deferred:**

- Broader ENTSO-G and GIE E2E coverage remains a future requirement.
- Full Elexon endpoint catalog remains a future requirement.
- New Elexon dataset promotion remains out of scope for this milestone.

**Technical Debt Incurred:**

- The first I4 live smoke run created ignored local `data/` artifacts before the config precedence fix; the verified behavior now uses temp roots.
- Milestone audit was not run because `gsd-sdk` is unavailable in this runtime; phase verification artifacts are the close-out evidence.

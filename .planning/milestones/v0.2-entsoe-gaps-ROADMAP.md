## Milestone Summary

**Key Decisions:**
- `replace_strict` chosen over soft replace for A-code mapping — raises on unknown codes, enforces data quality at transform time
- Unit-level dedup key `(timestamp_utc, unit_mrid)` chosen over zone-level for outages — matches spec's RegisteredResource semantics
- `ingested_at` added to all Phase 3 schemas retroactively — aligns with Phase 1/2 provenance pattern
- GAP-03b (wind_solar_forecast psrType codes B16/B18/B19) explicitly deferred — only GAP-03a (column rename) was in scope

**Issues Resolved:**
- BLOCKER-01: Phase 3 transformers were constructing doubled bronze paths (e.g., `entsoe/imbalance_prices/YYYY-MM-DD`) instead of `year/MM/DD` relative to `self.bronze_dir`
- GAP-07: All ENTSO-E price fields were named `price_gbp_mwh` despite data being denominated in EUR — corrected to `price_eur_mwh`

**Issues Deferred:**
- GAP-03b: wind_solar_forecast psrType codes (B16→solar, B18→wind_onshore, B19→wind_offshore) — not assigned to any gap-closure phase; track in backlog
- G1/G3/G4: No formal VERIFICATION.md files — self-checked via passing test suites and grep assertions in SUMMARYs

**Technical Debt Resolved:**
- `outages_generation` schema was zone-aggregate (`production_type`, `available_capacity_mw`) — now unit-level per spec
- All Phase 3 transformers lacked `ingested_at` — added across all 5 datasets

**Technical Debt Incurred:**
- GAP-03b psrType mapping unresolved (backlog)
- G1, G3, G4 lack formal VERIFICATION.md (process gap, not functional)

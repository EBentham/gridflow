# Milestone v0.5: ENTSOG Pipeline Validation

**Status:** SHIPPED 2026-05-04
**Phases:** J1-J4
**Total Plans:** Inline implementation

## Overview

Added ENTSOG as a fully validated pipeline source, from documented API endpoint
research through metadata-driven bronze ingestion, silver output, mocked tests,
fixture-backed bronze-to-silver checks, opt-in live API validation, and isolated
CLI smoke coverage.

## Phases

### Phase J1: ENTSOG Endpoint Catalog, Research, and Inventory Contract

**Goal:** Research the ENTSOG Transparency Platform API and make endpoint
coverage auditable across docs, source config, endpoint definitions, and silver
transformer registrations.

**Completed:** 2026-05-04

**Delivered:**

- Reviewed the ENTSOG TP API v2.1 manual, v3.0 manual, live API responses, and
  the `nhcb/entsog-py` client.
- Added `.planning/research/ENTSOG-ENDPOINT-RESEARCH.md`.
- Added `docs/entsog_endpoint_catalog.yaml`.
- Added inventory tests proving configured datasets, endpoint registry entries,
  and silver transformer registrations stay aligned.

### Phase J2: ENTSOG Connector Request Builder and Bronze Endpoint Coverage

**Goal:** Replace one-off physical-flow request logic with endpoint metadata for
all active ENTSOG datasets.

**Completed:** 2026-05-04

**Delivered:**

- Added a 33-dataset ENTSOG endpoint registry covering operational indicators,
  CMP/event, aggregated, tariff, UMM, and reference endpoint families.
- Corrected live request construction to use exact-case `/operationalData`,
  `timeZone=UCT`, exact-case indicators, and mandatory `pointDirection` filters.
- Updated `config/sources.yaml` so active ENTSOG datasets are available through
  normal pipeline commands.

### Phase J3: ENTSOG Silver Transformers and Mocked E2E Coverage

**Goal:** Ensure ENTSOG bronze responses can become deterministic silver parquet
across operational, CMP/event, tariff/UMM, and reference endpoint families.

**Completed:** 2026-05-04

**Delivered:**

- Preserved the specialised physical-flow transformer and GWh/day normalisation.
- Added generic ENTSOG JSON silver transformers for all other active datasets.
- Added tolerant ENTSOG datetime parsing for placeholders and mixed timestamp
  formats returned by live API payloads.
- Added a duplicate-normalised-column guard after live ENTSOG returned both
  `isCAMRelevant` and `isCamRelevant` for CMP auction premiums.
- Added mocked request-shape and fixture-backed bronze-to-silver tests across
  every active ENTSOG dataset.

### Phase J4: ENTSOG Live API-to-Silver Tests, CLI Smoke, and Close-Out Docs

**Goal:** Prove real ENTSOG public API responses can flow through bronze and
silver, while classifying documented no-data outcomes explicitly.

**Completed:** 2026-05-04

**Delivered:**

- Added opt-in live API-to-silver tests for all active ENTSOG endpoint families.
- Added isolated live CLI smoke tests for `gridflow pipeline entsog ...`.
- Verified real `cmp_auction_premiums` backfill for 2026-04-15 to 2026-04-16
  after fixing the duplicate-column collision.
- Updated planning docs, endpoint reference docs, and milestone state.

## Key Decisions

- ENTSOG v0.5 mirrors the ENTSO-E/Elexon validation pattern: inventory contract,
  mocked request-shape checks, fixture-backed bronze-to-silver tests, opt-in live
  API-to-silver tests, and isolated CLI smoke tests.
- The ENTSOG endpoint registry is the source of truth for request paths,
  response keys, parser families, default filters, and reference-vs-date-window
  behavior.
- Generic ENTSOG silver output is acceptable for endpoint coverage; domain-
  specific typed schemas are deferred until downstream gas modelling needs them.
- Live ENTSOG `404 No result found` and empty arrays are explicit no-data
  outcomes for narrow validation windows, not silent passes.

## Issues Resolved

- Corrected old lowercase `/operationaldata` and `timezone` assumptions to
  live-compatible `/operationalData` and `timeZone`.
- Added mandatory default `pointDirection` filters for operational and aggregated
  requests.
- Hardened datetime parsing against empty strings, `N/A`, and human-formatted
  metadata timestamps.
- Fixed duplicate snake_case column collisions from mixed ENTSOG field casing.

## Issues Deferred

- Add domain-specific ENTSOG silver schemas when downstream gold consumers need
  typed gas models.
- Add scheduled live ENTSOG endpoint monitoring outside the normal test suite.
- Apply the same validation pattern to GIE AGSI/ALSI in a future milestone.

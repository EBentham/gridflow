### Code Quality (1 item — low risk)

**outages_generation.py — missing schema validation guard**
`OutagesGenerationTransformer.transform()` does not import `EntsoeOutagesGeneration` or call `EntsoeOutagesGeneration(**sample)` on the first output row. All 15 peer transformers include this guard. Current column alignment is correct and tested (11 tests). Risk is that future schema changes (e.g., adding a validator or required field) will not be caught at transform time.

Fix: Add `from gridflow.schemas.entsoe import EntsoeOutagesGeneration` and a sample-row check before the final `return`.

### Open Audit Item (1 — deferred)

**GAP-03b — wind_solar_forecast psrType semantic mapping**
The original audit identified this as part of GAP-03 (medium severity). The gap-closure roadmap explicitly scoped only GAP-03a (column rename). The psrType mapping remains open. Downstream consumers of `wind_solar_forecast` silver will see raw codes (B16, B18, B19) not semantic strings. No model or gold view currently consumes this dataset, so no immediate breakage.

**Recommended action:** Add to project backlog as a future low-effort improvement.

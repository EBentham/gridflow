**Status:** ✅ SHIPPED 2026-05-09

**Total Plans:** 1

## Overview

Extends the Open-Meteo bronze and silver layers to provide the variable set and spatial coverage required for production-grade UK wind and solar forecasting downstream. After v0.11, silver carries hub-height wind (10m + 100m, ERA5 archive-verified), full irradiance components (GHI/DNI/DHI/GTI with UK fixed-tilt GTI params), cloud-cover decomposition (total, low, mid, high), snow variables, and derived air density. Location lists split into capacity-weighted wind (12 sites), solar (6 sites), and existing demand population centres (7 sites) = 25 sites total, 6 dataset families.

## Milestone Summary

**Key Decisions:**

- Hard rename with no aliases: `historical` → `historical_demand`, `forecast` → `forecast_demand`. Chosen for clean break; no compat shims in a local-only repo.
- Wave isolation abandoned: connector + schemas + transformers + tests combined into one atomic commit because tests import the symbols being deleted.
- Archive hub-heights: only 10m + 100m (verified live; 80/120/180m return all-null on ERA5 archive endpoint).
- Double-underscore separator `{dataset}__{location.name}` for unambiguous compound bronze keys.
- Air density band widened to `[0.95, 1.55]` from original `[0.95, 1.40]` after plan-time formula verification at joint extreme T=-30°C, P=1050 hPa → 1.504 kg/m³.
- Location centroids (ADR-020): approximate site centroids from public registers, not per-turbine NRO coordinates.
- `DATASET_VERSION` bumped 1.0.0 → 2.0.0 across all 6 transformer subclasses.

**Issues Resolved:**

- Serving client (`get_weather()`) queried deleted `silver_historical` view — fixed to `silver_historical_demand`.
- `scripts/run_all_sources.py` iterated `["historical","forecast"]` causing ValueError — fixed to 6 new names.
- Dead assert in `test_openmeteo_air_density.py` (always `True`) — removed.

**Issues Deferred:**

- F7.5-VAULT-01: Obsidian vault `quant-vault/30-vendors/open-meteo/` sync — requires `obsidian-vault` MCP server session. In-repo `docs/endpoints/open_meteo.md` is the version-controlled substitute.
- Workstream C (`minutely_15` AROME forecast resolution) — backlog, gated on AROME 2026 boundary verification.
- Open-Meteo ensemble endpoint (`/v1/ensemble`) — backlog.
- Wind/solar bronze backfill 2018-2025 (~52K location-days) — documented in F7.5-RESULTS.md, not executed.

**Technical Debt Incurred:**

- `BaseOpenMeteoTransformer.SCHEMA` ClassVar declared but `transform()` never validates against it — row validation against schema is skipped, mirroring the F0 pre-existing gap. Fix: wire schema validation in a follow-up (Option A from code review WR-03).
- `is_historical = dataset.startswith("historical")` URL routing is string-prefix brittle — future fix: add `archive: bool` field to `WeatherDatasetSpec`.

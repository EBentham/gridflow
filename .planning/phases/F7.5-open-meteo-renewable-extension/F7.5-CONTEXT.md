### Endpoint refactor sketch (`endpoints.py`)

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class WeatherLocation:
    name: str
    latitude: float
    longitude: float
    timezone: str = "UTC"

@dataclass(frozen=True)
class WeatherDatasetSpec:
    """Per-dataset endpoint spec consumed by OpenMeteoConnector."""
    locations: tuple[WeatherLocation, ...]
    hourly: tuple[str, ...]
    extra_params: tuple[tuple[str, str], ...] = ()  # frozen for the dataclass

DEMAND_LOCATIONS = (...)   # 7 sites
WIND_LOCATIONS = (...)     # 12 sites
SOLAR_LOCATIONS = (...)    # 6 sites

DEMAND_HOURLY_VARS = (...)
WIND_ARCHIVE_VARS = (...)        # excludes 80/120/180m
WIND_FORECAST_VARS = (...)       # includes 80/120/180m
SOLAR_HOURLY_VARS = (...)        # archive == forecast for solar

DATASET_SPECS: dict[str, WeatherDatasetSpec] = {
    "historical_demand": WeatherDatasetSpec(DEMAND_LOCATIONS, DEMAND_HOURLY_VARS),
    "forecast_demand":   WeatherDatasetSpec(DEMAND_LOCATIONS, DEMAND_HOURLY_VARS),
    "historical_wind":   WeatherDatasetSpec(WIND_LOCATIONS, WIND_ARCHIVE_VARS),
    "forecast_wind":     WeatherDatasetSpec(WIND_LOCATIONS, WIND_FORECAST_VARS),
    "historical_solar":  WeatherDatasetSpec(
        SOLAR_LOCATIONS, SOLAR_HOURLY_VARS,
        extra_params=(("tilt", "35"), ("azimuth", "180")),
    ),
    "forecast_solar":    WeatherDatasetSpec(
        SOLAR_LOCATIONS, SOLAR_HOURLY_VARS,
        extra_params=(("tilt", "35"), ("azimuth", "180")),
    ),
}

ARCHIVE_BASE_URL = "https://archive-api.open-meteo.com/v1"
FORECAST_BASE_URL = "https://api.open-meteo.com/v1"
```

Note `extra_params` is a tuple of `(key, value)` pairs not a dict, so `WeatherDatasetSpec` stays `frozen=True`. The connector materialises it back to a dict at request time.

async def _fetch_location(
    self, dataset: str, location: WeatherLocation,
    start: datetime, end: datetime,

        "timezone": "UTC",

    request_params.update(dict(spec.extra_params))

    ...

### Schema sketch (`schemas/weather.py`)

`DemandWeather` keeps the existing `WeatherObservation` field set plus `snowfall`, `snow_depth`, `air_density_kg_m3`. `WindWeather` is the demand set plus all hub-height and direction fields, gusts, cloud cover decomposition, and `dew_point_2m`. `SolarWeather` is GHI + DNI + DHI + GTI + cloud cover decomposition + snow + air-density. All hub-height and direction fields beyond `wind_speed_10m` and `wind_direction_10m` are typed `float | None = None` so the forecast/archive distinction (forecast may carry 80/120/180m) is permissive.

### Transformer split sketch

```python
class BaseOpenMeteoTransformer(BaseSilverTransformer):
    source = "open_meteo"
    HOURLY_VARS: ClassVar[tuple[str, ...]] = ()
    SCHEMA: ClassVar[type[BaseSchema]]
    DATASET_VERSION: ClassVar[str] = "2.0.0"
    DERIVE_AIR_DENSITY: ClassVar[bool] = False
    DERIVE_HDD_CDD: ClassVar[bool] = False

    def _pivot(self, raw_df: pl.DataFrame, locations: tuple[WeatherLocation, ...]) -> pl.DataFrame: ...
    def _add_derived(self, df: pl.DataFrame) -> pl.DataFrame:
        if self.DERIVE_AIR_DENSITY and {"surface_pressure", "temperature_2m"}.issubset(df.columns):
            df = df.with_columns(
                ((pl.col("surface_pressure") * 100.0) /
                 (287.05 * (pl.col("temperature_2m") + 273.15))).alias("air_density_kg_m3")
            )
        if self.DERIVE_HDD_CDD and "temperature_2m" in df.columns:
            df = df.with_columns([
                (pl.lit(15.5) - pl.col("temperature_2m")).clip(lower_bound=0).alias("hdd"),
                (pl.col("temperature_2m") - pl.lit(22.0)).clip(lower_bound=0).alias("cdd"),
            ])
        return df

class HistoricalDemandWeather(BaseOpenMeteoTransformer):
    dataset = "historical_demand"
    HOURLY_VARS = DEMAND_HOURLY_VARS
    SCHEMA = DemandWeather
    DERIVE_HDD_CDD = True
    DERIVE_AIR_DENSITY = True
    BRONZE_SIBLING_DATASETS = tuple(f"historical_demand__{loc.name}" for loc in DEMAND_LOCATIONS)
```

`ForecastDemandWeather`, `HistoricalWindWeather`, `ForecastWindWeather`, `HistoricalSolarWeather`, `ForecastSolarWeather` follow the same pattern. Wind transformers set `DERIVE_AIR_DENSITY = True`. Solar transformers do not (no `surface_pressure` in their fetch list).

### Threat model for the refactor

1. **Silent test-coverage drop on rename.** If `tests/fixtures/openmeteo/historical_london_response.json` is renamed but a test still references the old name, pytest will collect 0 tests rather than fail. Mitigation: keep the fixture file at the same path, only change the dataset string in tests; verify `pytest tests/unit/test_openmeteo.py -v` collects ≥ existing test count.
2. **Breaking config users not in this repo.** `config/sources.yaml` rename of `historical`/`forecast` blocks → `historical_demand`/`forecast_demand` could break user-customised local configs. Mitigation: document the rename in the `F7.5-RESULTS.md` migration section; vault `endpoints.md` updates also surface it.
3. **Pydantic field-permissiveness backfire.** All hub-height fields are `float | None` so wind silver doesn't fail validation when archive returns null — but a downstream consumer that depends on 100m always being non-null will break silently if they assume non-null. Mitigation: in vault `WindWeather` page, document explicitly which heights are guaranteed (10m, 100m on archive) and which may be null (80/120/180m). The schema is the contract; the docstring is the warning label.
4. **Air-density numeric edge cases.** If `surface_pressure` is in unexpected units (the API documents hPa but a future model change could differ), the derived field would be wrong by 100×. Mitigation: the property test asserts `air_density_kg_m3 ∈ [0.95, 1.40]` for synthetic rows in a documented temperature/pressure range; a unit-mismatch produces values 100× outside that band and fails loudly.
5. **GTI request failure on solar.** `tilt`/`azimuth` query params are only valid when `global_tilted_irradiance` is in the variable list. If the spec table is mis-edited so the params attach to a non-solar dataset, Open-Meteo returns a 400. Mitigation: a small contract test asserts `extra_params` is non-empty if and only if `"global_tilted_irradiance" in hourly`.
6. **Double-underscore parsing collision.** If a future location is named `east_anglia__norfolk` (already underscored), the double-underscore separator becomes ambiguous. Mitigation: a contract test asserts `"__" not in loc.name` for every location across the three lists.

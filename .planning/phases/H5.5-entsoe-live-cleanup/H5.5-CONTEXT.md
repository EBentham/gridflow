# H5.5 Context - ENTSO-E Live Cleanup

## Trigger

The credentialed live suite does not pass after H5:

```powershell
uv run --extra dev pytest -m live tests/integration/test_entsoe_live.py -v -rs
```

Observed first run: 12 failed, 46 passed, 6 deselected.

## Initial Findings

- `activated_balancing_qty` used `documentType=A83`, `processType=A16`,
  and `businessType=A95`; the live API rejects that combination. A follow-up
  probe without `processType` and with `businessType=A96` was also rejected for
  the default GB control area, so active all-dataset ingestion should not call
  A83 until a supported control-area strategy exists.
- Several balancing and unit datasets return HTTP 200
  `Acknowledgement_MarketDocument` payloads with `No matching data found` for
  the fixed GB date `2024-01-15`.
- `outages_generation` can return `application/zip` containing XML outage
  documents; the bronze/live test path expects XML files.
- Live `installed_capacity_units` payloads use flat tags such as
  `inBiddingZone_Domain.mRID`, `registeredResource.mRID`, and
  `generatingUnit_PSRType.psrType`, which the generic time-series parser must
  understand.
- Live outage payloads use `Available_Period` and
  `production_RegisteredResource.*` fields rather than the compact fixture
  shape.

## Guardrails

- Keep live tests opt-in and token-redacted.
- Do not weaken mocked E2E coverage; fixture-backed transforms still must
  produce rows.
- Treat genuine ENTSO-E no-data acknowledgements differently from malformed
  requests or parser regressions.
- Preserve H6-H8 roadmap scope; H5.5 is cleanup before new source batches.

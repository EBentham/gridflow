# H5.5-01 Summary - ENTSO-E Live Cleanup

## What Changed

- Added H5.5 planning and debug artifacts to capture the live-suite cleanup
  phase before H6.
- Reclassified `activated_balancing_qty` out of active `DOC_TYPES` and
  `config/sources.yaml` because ENTSO-E rejects A83 for the default GB control
  area as not allowed to be fetched via this service.
- Kept the A83 transformer/schema/tests available, while moving the catalog row
  to deferred H8 follow-up for a supported control-area strategy.
- Updated the ENTSO-E connector to expand `application/zip` responses into XML
  `RawResponse` objects so bronze writes `.xml` files and existing transformers
  can consume outage payloads.
- Extended the generic ENTSO-E time-series parser for live unit/outage variants:
  `inBiddingZone_Domain.mRID`, flat `timeInterval.start`,
  `Available_Period`, `registeredResource.*`,
  `generatingUnit_PSRType.psrType`, and `production_RegisteredResource.*`.
- Updated live tests to skip genuine ENTSO-E `No matching data found`
  acknowledgements for fixed-date transform checks, while still failing on
  invalid requests and command failures.
- Relaxed CLI live silver-directory assertions so no-data datasets do not create
  false negatives; bronze coverage remains strict for active datasets.
